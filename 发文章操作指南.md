# 发文章操作指南

这份文档放在 Hexo 主目录，方便以后按固定流程写文章、预览博客、同步到 GitHub。

## 1. 日常发文章的标准流程

以后发文章，按这 5 步走：

1. 新建文章
2. 写正文
3. 本地预览
4. 提交源码到 GitHub
5. 等 GitHub Actions 自动发布

---

## 2. 新建文章

在项目根目录打开终端：

```powershell
cd F:\Projects\hexo-jason-blog
```

然后执行：

```powershell
npx hexo new "文章标题"
```

例如：

```powershell
npx hexo new "如何搭建自己的技术博客"
```

执行后，Hexo 会在：

- `source/_posts/`

下面生成新的 Markdown 文件。

---

## 3. 编辑文章

新文章默认会出现在：

- `source/_posts/`

你直接打开对应 `.md` 文件写内容就行。

一个常用模板如下：

```markdown
---
title: 如何搭建自己的技术博客
date: 2026-04-10 23:30:00
categories:
  - 博客
tags:
  - Hexo
  - GitHub Pages
---

这里开始写正文。
```

### 常用字段说明

- `title`：文章标题
- `date`：发布时间
- `categories`：分类
- `tags`：标签

---

## 4. 本地预览

写完文章后，先本地看效果。

启动预览：

```powershell
npm run server
```

然后打开浏览器：

```text
http://localhost:4000
```

### 如果页面没刷新

可以先执行：

```powershell
npm run clean
npm run build
npm run server
```

---

## 5. 把文章同步到 GitHub

现在你的博客已经切成“源码仓库同步”模式，所以以后不要再把 `hexo deploy` 当成日常发布方式。

以后应该这样同步：

```powershell
git add .
git commit -m "docs(post): 新增文章《文章标题》"
git push origin main
```

例如：

```powershell
git add .
git commit -m "docs(post): 新增文章《如何搭建自己的技术博客》"
git push origin main
```

推上去之后：

- GitHub 仓库源码会更新
- GitHub Actions 会自动构建 Hexo
- GitHub Pages 会自动发布新站点

```powershell

```

### 第二步：本地预览

```powershell

```

```powershell

```

---

## 7. 如果只是改已有文章

如果你不是新建文章，而是修改旧文章，流程也是一样：

1. 直接编辑 `source/_posts/` 里的文章
2. 本地预览
3. 执行：

```powershell
git add .
git commit -m "docs(post): 更新文章《文章标题》"
git push origin main
```

## 8. 如果要新增页面

除了文章，你以后也可能会改这些页面：

- `source/about/index.md`
- `source/archives/index.md`
- `source/categories/index.md`
- `source/tags/index.md`

这些页面改完后，同样是：

```powershell
git add .
git commit -m "docs(page): 更新页面内容"
git push origin main
```

## 9. 常见问题

### 问题 1：本地和线上不一样

先执行：

```powershell
npm run clean
npm run build
npm run server
```

然后再看本地。

线上如果没立刻更新，通常是：

- GitHub Actions 还在跑
- GitHub Pages 有缓存

可以稍等几分钟再刷新。

### 问题 2：我改了文章但是网页没变

检查这几件事：

1. 是不是改错文件了
2. 有没有重新启动或刷新本地预览
3. 有没有 `git push origin main`
4. GitHub Actions 有没有执行成功

### 问题 3：文章文件在哪

默认都在：

- `source/_posts/`

## 11. 一个简单建议

以后每次发文章都先本地预览，再 push。  
这样可以避免格式、图片、分类、标签这些小问题直接上线上。
