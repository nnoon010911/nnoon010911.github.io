## 1. 项目目标与总体思路

本项目的目标是将训练完成的 DHS-YOLOv11 血小板检测模型封装为可交互展示的小程序系统，使用户能够在手机端查看测试集样本、上传血小板图像、获得检测结果图和预测计数结果。

整体采用前后端分离方案：

- 微信小程序前端负责页面展示、图片选择、图片上传、结果渲染和图片下载。
    
- FastAPI 后端负责加载 DHS-YOLOv11 权重、执行模型推理、统计检测框数量、生成检测结果图和返回 JSON 结果。
    
- Nginx 负责将公网 HTTPS 域名请求反向代理到后端服务端口。
    
- 服务器域名 `https://api.nnoon.top` 作为正式小程序访问后端的统一接口地址。
    

## 2. 数据与模型准备

首先完成血小板图像数据集构建。项目中包含测试集图像和对应标签文件，测试集用于展示模型在固定样本上的检测与计数效果。测试图片存放在后端可读取的测试图像目录中，标签文件用于统计人工标注数量。

随后完成 DHS-YOLOv11 模型训练。模型以 YOLOv11 为基础，引入 DySample 动态上采样模块和 HS-FPN 特征融合思想，用于改善显微小目标检测中的细节恢复和多尺度特征表达。训练完成后，将最佳权重文件 `best.pt` 作为小程序后端推理模型。

后端默认读取的权重路径为：

results/models/yolo11_dysample_hsfpn_n150/weights/best. pt

## 3. 后端服务构建

后端使用 Python 和 FastAPI 实现，主要文件包括：

backend/app. py  
backend/predictor. py

其中 `app.py` 负责提供 HTTP 接口和静态文件访问， `predictor.py` 负责模型加载、推理、计数、结果图绘制、缩略图生成和缓存处理。

后端主要接口如下：

|接口|作用|
|---|---|
| `GET /api/health` |检查后端运行状态、权重文件和样本数量|
| `GET /api/samples` |返回首页测试集样本列表|
| `GET /api/dataset-images` |返回数据集图片列表和缩略图地址|
| `GET /api/detect?image_name=xxx` |对测试集指定图片执行检测|
| `POST /api/detect/upload` |接收用户上传图片并执行检测|
| `/static/...` |提供测试图、上传图、结果图和缩略图访问|

后端推理参数主要包括：

imgsz = 1216  
conf = 0.25  
iou = 0.7  
max_det = 300  
device = cpu

检测完成后，后端通过检测框数量得到预测血小板数量。对于测试集样本，还会读取标签文件统计真实数量，并计算预测数量与真实数量之间的误差；对于上传图片，由于没有人工标注，只返回预测数量、推理耗时、原图地址和检测结果图地址。

## 4. 小程序前端构建

小程序前端位于：

miniapp/

主要页面包括：

|页面|功能|
|---|---|
| `pages/index/index` |首页，展示系统说明、数据集入口、上传检测入口和测试集样本|
| `pages/dataset/dataset` |数据集图片区，支持图片预览和下载|
| `pages/result/result` |结果页，展示原图、检测结果图、预测数量、真实数量、误差和耗时|

前端接口封装文件为：

miniapp/utils/api. js

小程序后端地址配置文件为：

miniapp/config. js

正式上线时配置为：

module. exports = {  
  backendMode: "cloud",  
  localBackendBaseUrl: "http://172.20.10.4:8010",  
  cloudBackendBaseUrl: "https://api.nnoon.top"  
};

其中 `backendMode: "cloud"` 表示正式环境下使用 HTTPS 域名访问后端。

## 5. 上传图片检测流程

用户在首页点击“上传图片并检测”后，小程序调用 `wx.chooseMedia` 选择本地图片。选择成功后，前端通过 `wx.uploadFile` 将图片上传到后端：

POST /api/detect/upload

后端接收图片后执行以下步骤：

1. 校验图片后缀是否为 `jpg` 、 `jpeg` 、 `png` 、 `bmp` 、 `tif` 或 `tiff` 。
    
2. 使用随机文件名保存上传图片到运行目录。
    
3. 调用 DHS-YOLOv11 模型执行推理。
    
4. 根据检测框数量统计预测血小板数量。
    
5. 使用 PIL 绘制检测框并生成结果图。
    
6. 生成原图和结果图缩略图，提升手机端加载速度。
    
7. 将检测结果以 JSON 返回给小程序。
    

小程序收到返回结果后，不再重复请求检测接口，而是直接跳转到结果页展示本次上传的检测结果。

## 6. 测试集样本检测流程

首页会请求：

GET /api/samples

后端返回测试集样本名称、真实标注数量、原图地址和缩略图地址。用户点击某个测试集样本后，小程序跳转到结果页，并根据图片名请求：

GET /api/detect? image_name=pic_xxx. png

后端从测试集目录中读取图片，调用模型执行检测，并读取对应标签文件统计真实数量。结果页展示以下信息：

- DHS-YOLOv11 预测数量
    
- 人工标注真实数量
    
- 计数误差
    
- 推理耗时
    
- 测试集原图
    
- 检测结果图
    

## 7. 数据集图片预览与下载

为了方便展示和复现实验流程，小程序增加了数据集图片下载区。用户可先进入数据集图片区浏览样本图像，点击图片进行预览，也可以将图片保存到手机相册，再返回首页上传该图片完成检测。

数据集图片列表由后端接口提供：

GET /api/dataset-images

后端同时提供数据集缩略图目录：

/static/dataset-thumbs/

该优化减少了手机端直接加载大图造成的等待时间，使首页和数据集页面加载更稳定。

## 8. 结果图展示与下载优化

结果页展示原始图像和 DHS-YOLOv11 检测结果图。为了提升展示效果，后端生成检测结果图时只保留图像本身和检测框，不再额外添加顶部白色文字栏，从而避免结果图出现多余白边。

结果页还增加了“点击下载结果图”入口，用户点击后小程序调用 `wx.downloadFile` 下载结果图，再通过 `wx.saveImageToPhotosAlbum` 保存到手机相册。正式版中该功能依赖 `downloadFile` 合法域名配置。

## 9. 局域网调试阶段

在正式域名可用之前，曾使用局域网和手机热点进行真机调试。此阶段需要保证电脑和手机处于同一网络，并将小程序后端地址配置为电脑局域网 IP，例如：

http://172.20.10.4:8010

同时后端不能只监听 `127.0.0.1` ，而应监听：

0.0.0.0:8010

Windows 防火墙需要放行后端端口，否则手机虽然连接同一热点，也无法访问电脑后端服务。该阶段主要用于快速演示和真机预览，但不适合正式上线，因为正式小程序必须使用 HTTPS 合法域名。

## 10. 服务器部署

正式上线前，将后端文件、模型权重、测试集、数据集图片、缩略图和运行依赖上传到服务器。服务器端后端目录为：

/srv/dhs/backend

后端通过 Python 虚拟环境运行，并监听 8010 端口。典型启动方式为：

cd /srv/dhs/backend  
nohup .venv/bin/python3 .venv/bin/gunicorn app: app -k uvicorn. workers. UvicornWorker -w 1 -b 0.0.0.0:8010 > runtime/gunicorn. out. log 2>&1 &

部署后先在服务器本机检查：

curl http://127.0.0.1:8010/api/health

确认返回 `status: ok` ，并确认 `weightsExists` 为 `true` 。

## 11. 域名、HTTPS 与 Nginx 配置

正式小程序不能使用 HTTP IP 地址访问后端，因此配置了接口域名：

https://api.nnoon.top

域名解析到服务器公网 IP 后，在 Nginx 中配置 HTTPS 反向代理：

https://api.nnoon.top -> http://127.0.0.1:8010

Nginx 负责 SSL 证书和外部 HTTPS 访问，FastAPI 后端仍然运行在服务器本地 8010 端口。配置完成后依次测试：

curl https://api.nnoon.top/api/health  
curl https://api.nnoon.top/api/samples  
curl https://api.nnoon.top/api/dataset-images

浏览器访问：

https://api.nnoon.top/api/health

能够看到 JSON 返回结果后，说明 HTTPS 后端已经可用。

## 12. 微信公众平台合法域名配置

进入微信公众平台，在“开发管理 -> 开发设置 -> 服务器域名”中配置：

request 合法域名: https://api.nnoon.top  
uploadFile 合法域名: https://api.nnoon.top  
downloadFile 合法域名: https://api.nnoon.top

对应关系如下：

- `request` ：加载测试集、加载数据集列表、请求检测结果。
    
- `uploadFile` ：上传本地图片进行检测。
    
- `downloadFile` ：下载数据集图片和检测结果图。
    

配置完成后，在微信开发者工具中关闭“不校验合法域名、web-view、TLS 版本以及 HTTPS 证书”，重新编译项目，确认正式域名校验条件下功能仍然可用。

## 13. 小程序上传、体验版与正式上线

本地功能测试完成后，在微信开发者工具中上传代码。上传时填写版本号和版本描述，例如：

DHS-YOLOv11 血小板检测与计数小程序，支持测试集样本浏览、数据集图片预览下载、上传血小板图像进行目标检测与计数，并展示原图、检测结果图、预测数量和推理耗时。

上传后先设置为体验版，在真机微信中测试完整流程：

1. 首页是否正常加载。
    
2. 测试集展示是否正常显示。
    
3. 数据集图片是否能预览和下载。
    
4. 上传图片检测是否成功。
    
5. 结果页是否显示预测数量、原图、检测结果图和推理耗时。
    
6. 结果图下载是否能保存到相册。
    
7. 是否存在合法域名、HTTPS 或接口超时报错。
    

体验版验证通过后，提交微信审核。审核通过后发布正式版，小程序即完成上线。

## 14. 上线后的最终功能

正式上线后，小程序实现了以下主要功能：

- 展示 DHS-YOLOv11 血小板检测与计数系统首页。
    
- 浏览测试集样本，并进入结果页查看检测效果。
    
- 浏览数据集图片，支持预览和保存到手机相册。
    
- 上传本地血小板图像并自动完成检测。
    
- 显示预测血小板数量、真实数量、计数误差和推理耗时。
    
- 展示原图和检测结果图。
    
- 支持下载检测结果图。
    
- 通过 `https://api.nnoon.top` 正式域名访问后端服务。
    

## 15. 常见问题与处理经验

### 15.1 手机扫码提示没有体验权限

这是小程序体验版权限问题。需要在微信公众平台把测试微信号加入体验成员，或者提交审核并发布正式版。正式版上线后，普通用户不再需要体验权限。

### 15.2 开发者工具能访问，手机访问失败

如果使用局域网地址，手机和电脑必须连接同一网络，后端必须监听 `0.0.0.0` ，并且 Windows 防火墙要放行端口。正式上线后应使用 HTTPS 域名，不再依赖热点和局域网 IP。

### 15.3 数据集图片或结果图加载慢

直接加载原图会比较慢，因此后端增加了缩略图生成逻辑。小程序优先加载缩略图，用户需要查看高清图或下载时再访问原图或结果图。

### 15.4 上传检测卡住

优先检查三点：后端服务是否正常运行， `uploadFile` 合法域名是否配置，服务器后端日志是否有推理异常。也要注意 CPU 推理耗时较长，上传检测接口超时时间需要适当设置。

### 15.5 HTTPS 域名打不开

依次检查 DNS 解析、服务器安全组、Nginx 443 端口监听、SSL 证书路径、反向代理地址和后端 8010 端口服务。若服务器本机能访问但外网不能访问，通常优先检查云服务器安全组和域名解析。

## 16. 总结

本项目从 DHS-YOLOv11 血小板检测模型出发，完成了模型推理服务封装、小程序前端页面开发、上传检测流程设计、测试集结果展示、数据集图片下载、服务器部署、HTTPS 域名接入、微信合法域名配置和正式版发布。最终形成了一个可在手机端直接使用的血小板检测与计数小程序，为毕业设计成果展示和后续应用扩展提供了完整实现基础。
